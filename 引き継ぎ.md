## 運用方針

- Codexは、ブランチ作成、実装、差分確認、ローカル状態確認までを担当する
- commit、push、PR作成、マージは原則としてユーザーが行う
- Codexがcommit、push、PR作成を代理するのは、ユーザーが明示的に依頼した場合だけ
- ドキュメントだけの変更は、ユーザー判断でPRを作らずmainへ直接反映してよい
- コード、DB、認証、クラウド設定、デプロイ挙動に影響する変更はPRを作る
- 固定サブエージェント名であるデービス、ガルシアは使わない
- agy CLIはレビュー補助に限定する
- agy CLIには実装、commit、push、PR作成、マージ、本番接続、本番DB接続を任せない
- agy CLIは通常レビューが緩くなる可能性があるため、使う場合は必ず`AGENTS.md`または`docs/codex-agent-team.md`の厳格レビュープロンプトを使う

agy CLIの確認済み実体:

```text
C:\Users\ユーザー\AppData\Local\agy\bin\agy.exe
```

新しいPowerShellまたはVS CodeターミナルでPATHが反映されていれば、次で呼べる。

```powershell

```

`agy` が見つからない場合は、フルパスで呼ぶ。

`--dangerously-skip-permissions` は、ユーザーが明示的に許可しない限り使わない。レビュー用途では原則不要。

## Git操作の基本

ユーザーがGit操作を行う場合は、`C:\Users\ユーザー\Documents\git.md` のテンプレートを使う。

このリポジトリ内で新しい作業を始めるときは、mainへ直接変更せず、作業ブランチを作る。

```powershell
git switch main
git pull origin main
git switch -c docs/update-handoff
```

すでにmain上で未コミット変更がある場合でも、次のように作業ブランチへ移してからcommitできる。

```powershell
git switch -c docs/update-handoff
git status --short
```

## 既知の前提

- Render無料枠はスリープするため、初回アクセスが遅いこと自体は異常ではない
- Neon無料枠の容量は現状の小規模なスケジュールデータでは十分余裕がある見込み
- ただし、正式運用前にはバックアップ、復元、無料枠継続可否を確認する
- 共通パスワードは入口制限であり、個人別の権限管理ではない
- 実在する会社名、人名、住所、電話番号、顧客名などはドキュメント、テスト、サンプルデータへ入れない
