# 配送・設置案件スケジュール管理システム

配送・設置案件をExcelで日付別に管理している現場を想定し、案件情報、担当者、時間帯、持参物、集合情報、車両情報などをWeb上で一元管理するためのポートフォリオ開発プロジェクトです。

現行業務では、共有Excel上で「誰が、いつ、どの案件を入れたか」は確認できますが、住所、持参物、同行有無、集合場所、車両、作業メモなどの詳細がセル内に収まりにくく、担当者への個別確認が発生しやすい状態です。本システムでは、スケジュール表から案件詳細へ自然に遷移できる構成にし、現場担当者と社員双方の確認コストを減らすことを目的とします。

## 開発目的

- Excel運用で発生している情報不足、確認漏れ、属人化を改善する
- 案件ごとの詳細情報を構造化し、後から確認しやすくする
- Java / Spring Boot / DBを中心に、SIer就活で説明しやすいWebアプリとして段階的に開発する
- 上流工程、設計、実装、テスト、改善の流れをポートフォリオとして示す

## 想定ユーザー

- 社員: 案件を登録し、担当者や作業内容を調整する
- 配送・設置担当者: 自分の担当案件と詳細情報を確認する
- 管理者: ユーザー、車両、品目などの基本情報を管理する

## MVPで目指す範囲

- 日付・時間帯ごとの案件スケジュール確認
- 案件の登録、編集、詳細確認
- 担当者、集合時間、集合場所、車両、持参物、作業メモの管理
- 案件ステータスの管理
- 最低限のユーザー権限

通知、地図連携、ルート最適化、Excel取込、外部カレンダー連携などは将来拡張として扱います。

## 技術方針

初期段階では技術を増やしすぎず、以下を中心に検討します。

- Java
- Spring Boot
- RDB
- HTMLテンプレートまたはシンプルなフロントエンド
- JUnitを中心とした自動テスト

詳細な技術選定は、要件整理と画面・DB設計の初期案を固めた後に決定します。

## 開発ルール

- mainブランチへ直接変更しない
- 作業ごとにブランチを作成し、Pull Requestで変更する
- 実在する会社名、人名、住所、電話番号、車両番号、顧客名などは使わない
- サンプルデータは必ず架空データにする
- MVPと将来拡張を分けて管理する
