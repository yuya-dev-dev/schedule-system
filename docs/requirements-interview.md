# 要件ヒアリング議事録

## このドキュメントの目的

顧客ヒアリングで出た要望、決定事項、上書きされた案、未確認事項を残す。会話の文脈が圧縮されても、ここを読めば最新の要件判断を復元できる状態にする。

このドキュメントでは、会話途中で考えが変わった箇所について、古い案を採用しないように明示する。

## 前提

- 顧客役: 現場で配送・設置案件を確認する担当者
- ベンダ役: 要件定義を支援するシニアPM
- 対象業務: コーヒーサーバー、ウォーターサーバーなどの配送・設置・回収・交換に関わるスケジュール管理
- 注意: 実在する会社名、人名、住所、電話番号、車両番号、顧客名は使わない。サンプルは必ず架空データにする。

## 最新の採用方針

### 1. システム化の目的

- 現行Excelの「ぱっと見で日付、時間帯、予定の埋まり具合が分かる」UIは気に入っている。
- そのため、Web版でもExcelに近いスケジュール表UIを重視する。
- 最大の課題は、Excelセル内に案件詳細を十分に書けないこと。
- Web版では、スケジュール表の見やすさを残しつつ、案件ごとの詳細入力フォームを用意する。
- 朝の確認、電話、LINE、口頭確認の手順を減らし、聞き漏れを防ぐ。
- 通知機能は便利だが、現時点では優先度は低い。

### 2. 現行の入力者と運用

- 入力する社員は日によって変わる。
- 関係する社員はおよそ10人前後。
- 1日に実際にスケジュールを埋める社員は3人程度。
- 複数人が入れたいタイミングで自由に入力する。
- 入力者と案件依頼者は同じ。
- 早い者勝ちで予定が埋まる。
- 最終承認者や確定担当者は置かない。
- 入力した時点で依頼として扱う。

### 3. スケジュール一覧UI

- 現行Excelに近い、日付と時間帯を軸にした表形式を採用する。
- 現行Excelは1セル1時間だが、新システムでは1セル30分とする。
- 1セルに入る案件は1件のみとする。
- 1案件は複数の30分セルにまたがることがある。
- サイトを開いた時点の現在年月の月間スケジュール一覧を自動表示する。
- 現行Excelの月別シートに近い骨組みを採用する。
- 月タブまたは前月・翌月操作で別月のスケジュール一覧へ切り替えられる。
- たとえば6月28日にサイトを開いた場合は6月版を表示し、7月予定を入れたい場合は月タブから7月版へ移動する。
- 月別スケジュール一覧の任意の日付内のセルをクリックすると入力フォームへ遷移する。
- クリックしたセルの時間帯は、入力フォームの時間範囲を固定しない。
- セルクリックは「日付を選ぶ入口」として扱う。
- フォームで入力された開始時間、終了時間に基づいて、該当する30分セルが同じ色で表示される。

### 4. スケジュール表への反映条件

スケジュール一覧へ案件を表示する条件は、次の3つがそろっていること。

- 依頼者名が入力されている
- 開始時間が入力されている
- 終了時間が入力されている

この条件を満たす場合、該当日の開始時間から終了時間までの30分セルに色を付け、依頼者名と作業種別などの概要を表示する。

依頼者名、開始時間、終了時間のいずれかが未入力の場合、スケジュール一覧には反映しない。

### 5. 時間範囲の重複チェック

- 同じ日の既存案件と時間範囲が重なる入力は無効にする。
- 重複時は保存せず、注意表示を出す。
- 表示例: `その時間はすでに埋まっています`
- 大規模な同時編集や入力衝突対策はMVPでは考慮しない。
- ただし、保存時または自動保存時の時間重複チェックは必要。

例:

- 既存案件: 9:00-11:00
- 新規入力: 10:00-12:00
- 結果: 重複しているため反映しない

### 6. 入力フォームの保存方針

- 確定ボタンは置かない。
- 理由は、入力後に確定ボタンを押し忘れ、ページ移動で入力内容が消える事故を避けるため。
- 現行Excelに近く、入力した内容が反映される軽い操作感を重視する。
- MVPでは、完全リアルタイム保存か、自動保存か、フォーム離脱時保存かは実装設計時に検討する。
- ただし、明示的な確定ボタンを押さないと予定に反映されない方式は採用しない。

入力フォームに置く主なボタン:

- 依頼キャンセルボタン
- 前のページに戻るボタン

依頼キャンセルボタンは押し間違え防止のため、二重確認を表示する。

### 7. ステータス管理

MVPでは案件ステータスを持たない。

採用しないステータス:

- 確定
- 未確認
- 確認済み
- 作業中
- 完了
- キャンセル済み

理由:

- 1日の案件数は多くても数件程度で、完了管理の必要性が低い。
- 確認済みを置くと、その後に社員が変更・追記しづらくなる。
- キャンセル済みという表示は不要で、依頼キャンセルにより該当セルが空けば一覧上で判断できる。
- Excelライクな軽い運用を優先する。

キャンセル操作のMVP方針:

- 依頼キャンセルボタンで入力内容を削除または一覧非表示にする。
- 一覧上は該当セルの色と依頼者名が消える。
- キャンセル履歴の保持は将来拡張とする。

### 8. 作業種別

MVPの作業種別は次の4つとする。

- 設置
- 回収
- 交換
- 配達

### 9. 必須項目

MVPの入力フォームで必須にする項目:

- 依頼者名
- 開始時間
- 終了時間
- 作業種別
- 依頼内容
- 住所
- 現場到着希望時間
- 一人作業 / 同行あり

補足:

- 日付はフォームで選ばせない。スケジュール表でクリックした日付により決まる。
- 簡易案件タイトルは必須項目にしない。
- 台数、機種、内容物の詳細は独立項目にしない。
- 依頼内容という大きめの入力欄にまとめて記入する。

### 10. 条件付き必須項目

同行ありを選んだ場合のみ必須にする項目:

- 集合場所
- 出発時間
- 使用車両

理由:

- 同行なしの場合は、基本的に担当者本人の車を使う想定。
- 同行ありの場合は、集合場所、出発時間、使用車両がないと当日確認が必要になる。

### 11. 任意項目

MVPで任意項目として用意するもの:

- 出庫要否
- 備考

出庫要否を任意にする理由:

- 先に出庫して用意されている場合がある。
- 必須ではないが、分かると当日の準備がしやすい。

備考にまとめる内容:

- 搬入口
- 受付方法
- 入り方
- 現地で連絡する相手
- 警備員への受付
- その他注意事項

これらは独立項目にしない。現場で対応できる情報であり、項目を増やしすぎると入力負荷が上がるため。

### 12. ログイン・権限管理

MVPではログイン処理や個人識別を行わない。

採用方針:

- 誰もが同じ権限で参照・編集できる掲示板のようなシステムにする。
- 社員、配送・設置担当者、管理者のような権限差は設けない。
- ログイン、ユーザー管理、権限管理はMVPに含めない。

理由:

- 現行Excelも全員が同じように編集できる運用である。
- ログインや権限管理を入れると、MVPの操作や説明がややこしくなる。
- 今回の主目的は、Excelに近い一覧性を残しながら詳細情報を入力できるようにすることである。

### 13. 編集権限

MVPでは全員が同じ権限なので、配送・設置担当者も案件を編集できる。

ただし、実運用上は配送・設置担当者が積極的に編集する理由は少ない。受託者である担当者が内容を理解できればよく、不明点があれば依頼者へ連絡する。

多少のヒューマンエラーは許容する。ここは現行Excelと同じ軽い運用を優先し、細かい権限や承認機能は入れない。

### 14. 依頼者名の入力方式

MVPでは依頼者名は自由入力とする。

採用理由:

- 現行Excelに近く、入力が軽い。
- ログインや社員マスタを導入しない方針と整合する。
- 表記ゆれは起こり得るが、MVPでは許容する。

採用しない案:

- 依頼者名の自動入力
- 社員マスタからの候補選択

これらは、ログインや利用者識別が必要になった場合の将来拡張として扱う。

## 上書き・破棄された案

会話途中で出たが、最新方針では採用しない案を明示する。

| 古い案 | 最新方針 |
| --- | --- |
| 通知をMVPの重要機能として扱う | 通知は優先度低め。MVP必須ではない |
| 確定ボタンを置く | 確定ボタンは置かない |
| 入力した時点で確定ステータスにする | ステータス管理自体をMVPでは持たない |
| 確認済みステータスを持つ | 持たない。後から編集しづらくなるため |
| 完了済みステータスを持つ | 持たない。案件数が少なく不要 |
| キャンセル済みステータスを持つ | 持たない。キャンセル時は一覧から表示を消す |
| 依頼者名だけで一覧反映する | 依頼者名、開始時間、終了時間がそろった場合に反映する |
| クリックしたセルの時間帯をフォーム時間に固定する | セルクリックは日付選択のみ。時間範囲はフォームで入力する |
| 簡易案件タイトルを必須にする | 必須にしない |
| 台数、機種を個別項目にする | 個別項目にせず、依頼内容へまとめる |
| 試運転要否を項目化する | 項目化しない |
| 搬入口、受付、現地連絡先を個別項目にする | 個別項目にせず、備考にまとめる |
| ログイン画面をMVPに含める | 含めない |
| ユーザー管理画面をMVPに含める | 含めない |
| 管理者、社員、担当者で権限を分ける | 分けない。全員同じ権限にする |
| 依頼者名を自動入力にする | 自由入力にする |
| 依頼者名を候補選択にする | MVPでは採用しない |
