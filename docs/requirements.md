# 要件定義

## 目的

配送・設置案件を共有Excelで管理している現場を想定し、案件の予定と詳細情報をWebアプリ上で一元管理する。現場担当者が出勤後に複数の社員へ個別確認している情報を、案件詳細画面で確認できる状態にする。

## 現行業務の整理

現行Excelは、日付ごとの列と時間帯ごとの行を使い、案件を表形式で配置している。色分けや矢印により、時間帯や作業の継続を表現している。

ただし、Excelセルに書ける情報量が限られるため、次のような情報が分散しやすい。

- 作業先の住所
- 集合時間と集合場所
- 同行者の有無
- 使用車両
- 依頼内容
- 回収物や設置物の詳細
- 当日の注意事項
- 依頼者や確認先

## 現行業務の課題

- セル内に情報を書き切れず、案件詳細が不足する
- 担当者が社員へ個別にメールや電話で確認する必要がある
- 変更や追記が一覧から分かりにくい
- 同じ時間帯に複数案件が重なった場合、負荷や移動の見通しが立てづらい
- 車両や依頼内容の詳細が見えにくい
- 過去案件の検索や振り返りがしにくい
- Excelの自由入力により、表記ゆれや確認漏れが発生しやすい

## 対象利用者

| 利用者 | 主な目的 |
| --- | --- |
| 社員 | 案件を登録し、担当者、時間帯、作業内容を共有する |
| 配送・設置担当者 | 入力された案件の詳細を確認する |
| 全員共通 | ログインや権限差を設けず、同じ権限で参照・編集する |

## 提供価値

- スケジュールと案件詳細を紐づけ、確認作業を減らす
- 入力項目を構造化し、情報の抜け漏れを減らす
- 住所、到着希望時間、同行情報、車両、備考を確認しやすくする
- ログインや権限管理を入れず、現行Excelに近い軽い運用を保つ
- SIer就活用ポートフォリオとして、要件定義からテストまで説明できる構成にする

## MVPの範囲

最初の開発では、細かなUIを作り込む前に、入力から一覧反映まで実際に動く最小版を初期MVPの完成地点とする。

現行Excelが持つ月単位の一覧性、日付・曜日の見出し、時間帯別の予定表示、連続時間の色分けと矢印は、最低限の業務機能として引き継ぐ。ただし、Excel上で一定していない時間区切りは引き継がず、8:30-17:30を30分単位で表示する。案件セル内の会社名、機械名、住所なども引き継がず、依頼者名、継続矢印、案件色、`＊未入力` だけを表示する。

- 月間スケジュール一覧を表示できる
- 対象月のすべての水曜日・金曜日を日付列として自動生成し、日付順に表示できる
- 合意した最小限の項目を持つ案件入力フォームを表示できる
- フォームの入力内容を自動保存できる
- 依頼者名、開始時間、終了時間がそろった案件を月間一覧へ反映できる
- 同じ日の既存案件と時間範囲が重なる入力を無効にし、注意表示を出せる
- 二重確認後に案件を物理削除し、該当時間帯を未入力へ戻せる
- 日付を休みに設定し、その日の案件入力を禁止できる
- 祝日などの不要な日付列を月間一覧から削除できる
- 案件の入力内容を別の勤務日の新規フォームへコピーできる
- サイト起動時に現在月の一覧を自動表示できる
- 現在月を基準とした前月・当月・翌月のタブを表示し、対象月を切り替えられる
- iPhoneで一覧と入力フォームを正常に表示し、スクロール、拡大して確認できる

左上の年月選択、サンプルデータ投入、細かなUI調整は、初期MVPの完成条件に含めない。その他の合意済み機能は、初期MVPに含めるかを個別に確認しながら段階的に実装する。

### 機能要件

| ID | 機能 | 内容 |
| --- | --- | --- |
| FR-001 | スケジュール一覧表示 | 現在月を自動表示し、その月の水曜日・金曜日と30分単位の時間帯を軸に案件を一覧表示できる |
| FR-002 | 案件入力 | 全利用者が依頼者名、時間範囲、作業種別、依頼内容、現場住所などを入力できる |
| FR-003 | 案件編集 | 登録済み案件の日時、依頼内容、同行情報、備考などを編集できる |
| FR-004 | 案件詳細表示 | 案件ごとの現場住所、依頼内容、集合情報、車両、備考を確認できる |
| FR-005 | 同行情報管理 | 同行ありチェックを付けた場合のみ、集合場所と出発時間を入力できる |
| FR-006 | 車両情報管理 | 同行ありチェックを付けた場合のみ、使用車両を入力できる |
| FR-007 | 時間重複チェック | 同じ日の既存案件と時間範囲が重なる入力を無効にできる |
| FR-008 | 検索・絞り込み | 日付、依頼者名、作業種別で案件を絞り込める |
| FR-009 | 依頼キャンセル | 二重確認後に案件データを物理削除し、該当セルを未入力扱いに戻せる |
| FR-010 | 共通編集権限 | ログインなしで、全員が同じ権限で参照・編集できる |
| FR-011 | セル表示制御 | 先頭セルに依頼者名と＊未入力表示、2セル目以降に矢印、案件単位の色を表示できる |
| FR-012 | セルクリック分岐 | 空白セルは新規入力フォーム、入力済みセルは該当案件の確認・編集フォームへ遷移できる |
| FR-013 | 休み設定 | 水曜日・金曜日を日付単位で休みに設定し、一覧表示を残したまま案件入力を禁止できる |
| FR-014 | 日付列非表示 | 祝日などの不要な日付列を月間一覧から削除できる |
| FR-015 | 案件コピー | 既存案件の日付以外の入力内容を、別の勤務日の新規案件フォームへコピーできる |

### 非機能要件

| ID | 項目 | 内容 |
| --- | --- | --- |
| NFR-001 | 操作性 | Excelに近い時間帯別の見方を残しつつ、案件詳細へすぐ移動できる |
| NFR-002 | 保守性 | Java / Spring Bootの標準的な構成で、層を分けて実装する |
| NFR-003 | 拡張性 | 通知、地図連携、履歴管理などを後から追加しやすい構造にする |
| NFR-004 | 情報管理 | 実在の個人情報や顧客情報をサンプルデータに含めない。MVPではログインを置かないため、個人情報を入れすぎない |
| NFR-005 | 端末対応 | PC入力を主対象とし、iPhoneではスクロール、拡大によりスケジュール一覧と案件詳細を正常に確認できる |
| NFR-006 | 可用性 | ポートフォリオ段階では単一アプリ構成とし、停止時の復旧手順を明確にする |
| NFR-007 | 性能 | 一般的な月次スケジュール件数でストレスなく一覧表示できる |
| NFR-008 | テスト容易性 | ドメインロジックとサービス層を自動テストしやすくする |

## 将来拡張

MVP後に検討する機能は次の通り。

- ログイン、利用者識別、権限管理
- 案件ステータス管理
- 案件変更時のメール、チャット、アプリ通知
- Excelファイルからの案件取込
- ExcelまたはCSVへの出力
- Googleカレンダーなど外部カレンダー連携
- 地図表示、移動時間計算、ルート最適化
- 添付ファイル管理
- 作業完了報告、写真アップロード
- 承認フロー
- 詳細な監査ログ
- スマートフォン向けUIの強化

## 業務ルール初期案

- 案件には開始予定日時と終了予定日時を必須で持たせる
- サイトを開いたときは現在年月の月間スケジュール一覧を表示する
- 対象月に含まれるすべての水曜日・金曜日を日付列として自動生成し、日付順に並べる
- 水曜日・金曜日は祝日でも既定では表示し、必要に応じて日付列を削除できる
- 日付見出しからその日を休みに設定できる
- 既存案件がある日を休みに設定した場合、その日の全案件を物理削除し、復元用データや履歴は保持しない
- 休みの日は先頭の時間セルに `休み` と表示し、その日の全時間セルを未入力状態とは異なる専用色で表示する
- 休みの日の空白セルをクリックしても案件入力フォームへ遷移しない
- 休みの日を対象とする案件入力・編集項目は入力不可とする
- 案件入力・編集フォーム上部に `この入力内容をほかの日時にコピーする` 操作を配置する
- コピー操作後はコピー先選択画面を開き、日付の直接入力または簡易スケジュール表のクリックでコピー先を選べる
- 簡易スケジュール表では水曜日・金曜日と既存案件の時間帯を確認でき、休み・非表示の日は選択できない
- コピー時は元案件を残し、作業日と案件IDを除く入力値を新規フォームへ引き継ぐ
- 開始時間・終了時間もコピーし、移動先の日付にある既存案件と通常どおり重複チェックする
- 重複した場合はコピー先案件を保存せず、コピー値をフォームに残したまま `その時間はすでに埋まっています` と表示する
- 月タブは現在月を基準に前月・当月・翌月の3か月分を表示する
- 前月・当月・翌月以外の月は、スケジュール一覧画面左上端の年月選択から年度と月を選んで移動できる
- スケジュール表の表示時間帯は8:30-17:30固定とし、30分単位で表示する
- 案件の開始時間・終了時間は8:30-17:30内のみ入力できる
- スケジュール一覧への反映は、依頼者名、開始時間、終了時間がそろっている場合に限る
- 依頼者名、開始時間、終了時間がそろっているが他の必須項目が不足している場合、先頭セルに `＊未入力` と表示する
- 入力・編集フォームでは、未入力の必須項目を赤文字で項目名付き表示する
- `＊未入力` が表示されている案件でも、依頼者名、開始時間、終了時間がそろっている場合は時間枠を確保済みとして扱う
- 入力欄から離れたタイミングで自動保存する
- 前のページに戻る操作をした場合も未保存内容を保存する
- 複数セルにまたがる案件では、先頭セルに依頼者名、2セル目以降に矢印を表示する
- 色はシステム側で自動割り当てする。基本案は5色程度の固定パレットだが、よりシンプルな実装方法がある場合は採用してよい
- 前後の案件で同じ色が連続せず、キャンセル後も案件の時間範囲を見分けやすいことを優先する
- 同じ案件の時間範囲は同じ色で表示する
- セル内には作業種別や依頼内容を表示せず、詳細は入力・編集フォームで確認する
- 作業種別によってフォーム項目は切り替えず、細かい違いは依頼内容にまとめて入力する
- 空白セルをクリックした場合は、その日付の新規入力フォームへ遷移する
- 入力済みセルをクリックした場合は、そのセルに対応する既存案件の入力・編集フォームへ遷移する
- 空白セルから新規入力する場合も、クリックした時間帯で開始時間・終了時間を固定しない
- 同じ日の既存案件と時間範囲が重なる入力は無効にする
- 同行ありチェックを付けた場合のみ、集合場所、出発時間、使用車両の3項目を動的に表示し、必須にする
- 同行ありの場合の条件付き必須項目が不足している場合も、通常の必須項目と同じく `＊未入力` と赤文字警告の対象にする
- 同行ありチェックが付いていない場合、集合場所、出発時間、使用車両はフォーム上に表示せず、入力・必須判定の対象にしない
- 現場住所や顧客情報は案件詳細に集約する
- 現場住所は自由入力の大きめのテキスト欄とし、郵便番号や都道府県などには分割しない
- 現場到着希望時間は自由入力とし、時刻だけでなく `午前中`、`13時頃`、`時間指定なし` のような表現も許可する
- 搬入口、受付方法、現地担当者への連絡、注意事項などは備考欄に入力する
- 備考欄には `例: 現地担当者への連絡、注意事項等あれば` のような補助テキストを表示する
- 確認済み、完了などの案件ステータス管理はMVPでは行わない

## 対象外

MVPでは次を対象外とする。

- 給与、勤怠、労務管理
- 実在顧客データの移行
- 外部サービスとの本格連携
- 高度な配車最適化
- 請求、売上、在庫管理
- ログイン、権限管理、ユーザー管理
- 確認済み、完了、キャンセル済みなどのステータス管理
- iPhone専用の今日表示、選択日表示などの簡易表示

## 個人情報・機密情報の扱い

設計書、サンプルデータ、テストデータには、実在する会社名、人名、住所、電話番号、車両番号、顧客名を使わない。必要な場合は、次のような架空表現に置き換える。

- 顧客名: サンプル株式会社、テスト商事
- 担当者名: 社員A、担当者B
- 住所: 愛知県名古屋市サンプル区1-2-3、愛知県豊田市サンプル町4-5
- 車両: 車両A、車両B
- 電話番号: 000-0000-0000

サンプルデータは、1日あたり3件程度の案件を基本にし、作業種別は設置、回収、交換、配達を一通り含める。現場住所の市名は愛知県内に限定するが、番地、建物名、顧客名、担当者名はすべて架空データにする。

## 成功条件

- 担当者が当日の案件と詳細情報をWeb上で確認できる
- 社員が案件入力時に必要情報を入力できる
- Excelセルに収まらない情報を案件詳細として管理できる
- MVPと将来拡張の境界が説明できる
- 要件、画面、DB、テスト方針が一貫している
