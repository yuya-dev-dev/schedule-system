# テスト方針

## 目的

配送・設置案件のスケジュール管理において、日時、時間範囲、下書き、同行条件、一覧反映条件、同時登録などの業務ルールを安定して守る。ポートフォリオとして、実装だけでなく品質保証の考え方も説明できる状態にする。

フェーズ4で実行するケースID、優先度、環境、E2E、復旧、端末試験、合格基準は[フェーズ4 テスト設計](phase4-test-design.md)に定義する。

## 基本方針

- 重要な業務ルールは自動テストで守る
- 画面表示だけに依存せず、ドメインロジックとService層を重点的にテストする
- 自動保存、動的フォーム、下書き再開などの主要ブラウザ操作は、限定的なE2Eテストでも確認する
- 最小縦切りの完成時点で、空白セルから入力、保存、一覧反映、再表示までのスモークE2Eを1本先行する
- 機能実装と、その機能を守る単体・結合テストは原則として同じPRへ含める
- UIとUXの評価は、対象端末での手動確認と利用者の試運転で行う
- テストデータには実在する会社名、人名、住所、電話番号、車両番号、顧客名を使わない

## テストレベル

| レベル | 対象 | ツール候補 | MVP |
| --- | --- | --- | --- |
| 単体テスト | ドメイン、バリデーション、Service | JUnit 5, AssertJ | 対象 |
| Repositoryテスト | DBアクセス、楽観ロック | Spring Boot Test | 対象 |
| Controllerテスト | HTTPリクエスト、画面遷移 | MockMvc | 対象 |
| 結合テスト | 保存、下書き、重複判定、キャンセル | Spring Boot Test | 対象 |
| E2Eテスト | 最小縦切り、自動保存、動的項目、下書き再開 | Playwrightなど | 縦切り時に1本先行し、主要経路へ拡充 |
| 負荷テスト | 一覧表示性能 | JMeterなど | MVP後 |

## TDDを適用する領域

- 終了時間は開始時間より後であること
- 時刻が30分単位かつ8:30-17:30内であること
- 作業種別ごとの一覧反映条件がそろうまで `DRAFT` として扱うこと
- 時間重複した入力を理由付きの `DRAFT` として保持すること
- 下書きは時間枠を確保しないこと
- 通常案件へ移る際に既存案件との重複を拒否すること
- 同時登録では先着案件だけを `PUBLISHED` とし、後続を理由付き `DRAFT` とすること
- 同一案件の古いバージョンからの上書きを拒否すること
- 同行ありチェック時、集合場所と出発時間だけが条件付き必須となり、現場住所もしくは会社名が任意になること
- 同行解除時に3項目の保存値を削除すること
- 入庫、商品管理では依頼者名を任意とし、時間範囲と作業種別だけで不足警告を出さないこと

## 主要テストケース

### 自動保存と下書き

| ケース | 期待結果 |
| --- | --- |
| 何も入力せずフォームを離れる | 空の案件レコードを作らない |
| 最初の入力欄から離れる | 入力値が下書きとして保存される |
| 依頼者名だけ入力する | 一覧へ反映されず、下書き一覧へ表示される |
| 空白文字だけを入力する | 未入力として扱われ、空レコードは作成されない |
| 前のページに戻る | 未保存内容を保存してから一覧へ戻る |
| 下書き一覧を開く | 作業日、入力済み依頼者名、最終更新日時、下書き理由が表示される |
| 複数月に下書きがある | 作業日が当日以降である全月の下書きが最終更新日時の新しい順に表示される |
| 依頼者名が空の下書きを表示する | `依頼者名未入力` と表示される |
| `入力を再開` を押す | 保存済みの入力値を持つフォームが開く |
| 下書き削除を確定する | 下書きレコードが物理削除される |
| 作業日当日に下書き一覧を開く | 対象下書きが残り、入力を再開できる |
| 作業日の翌日に下書き一覧を開く | 対象下書きが自動で物理削除され、一覧に表示されない |
| 通常案件の作業日が過ぎる | 自動削除されず保持される |
| 下書きで開始・終了時間を入力する | 時間枠はまだ確保されない |
| 通常作業で依頼者名と時間範囲をそろえる | 重複確認後、`PUBLISHED` として一覧へ反映される |
| 依頼者名と時間範囲だけを入力する | 一覧へ反映され、先頭セルに `＊未入力` が表示される |
| 入庫、商品管理で時間範囲と作業種別をそろえる | 依頼者名が空でも `PUBLISHED` として一覧へ反映される |
| 一覧反映条件をそろえた時点で時間が埋まっている | `DRAFT` の値を保持し、下書き理由と競合時間を表示する |

### 案件入力・編集

| ケース | 期待結果 |
| --- | --- |
| 開始時間と同じ、または前の終了時間を入力する | エラーとなり通常案件へ移らない |
| 8:30より前または17:30より後を入力する | エラーとなる |
| 30分単位ではない時刻を入力する | エラーとなる |
| 設置、回収、交換、配達で詳細必須項目が不足する | 一覧の先頭セルに `＊未入力` を表示する |
| 未入力項目がある案件フォームを開く | フォーム上部と該当項目付近に不足項目名を赤字表示する |
| 必須項目を表示する | 項目名の先頭付近に赤字で `＊必須` を表示する |
| 顧客先到着希望時間に `午後かつ17時まで` を入力する | 自由入力として保存できる |
| 同行ありチェックを付ける | 集合場所、出発時間、使用車両の指定を表示し、前2項目だけを条件付き必須にする |
| 同行ありチェックを付ける | 現場住所もしくは会社名の `＊必須` 表示と入力必須制約を外す |
| 使用車両の指定を表示する | `指定がなければ空欄` のプレースホルダーを表示し、空欄でも保存できる |
| 同行ありチェックを外す | 3項目を非表示にし、保存値を削除する |
| 出庫状態の初期表示 | `未回答` が選択されている |
| 出庫状態を変更する | `出庫が必要` または `出庫済み` を保存できる |
| 入庫、商品管理で時間範囲と作業種別だけ入力する | 依頼者名が空でも `＊未入力` と不足警告を表示しない |
| 同じ案件を古い画面から更新する | 楽観ロックエラーとなり、入力値を保持して最新情報の再読込を促す |
| 入力欄から離れて保存する | `保存中...` の後に `保存済み` を表示する |
| 自動保存に失敗する | 入力値を保持し、赤字エラーと `再試行` を表示する |
| 通常作業の詳細入力後に入庫・商品管理へ変更する | 削除対象を確認し、承認後に新しい作業種別、依頼者名、時間範囲以外の詳細を削除する |
| 案件フォームを開く | 上部に対象年月日と曜日を表示し、日付を変更できない |

### 時間重複と同時登録

| ケース | 期待結果 |
| --- | --- |
| 9:00-11:00の既存案件へ10:00-12:00を登録する | 重複エラーとなる |
| 9:00-11:00の既存案件へ11:00-12:00を登録する | 登録できる |
| 2利用者が同じ空き時間へ同時登録する | 先にコミットした1件だけを `PUBLISHED` とし、後続を `DRAFT` として保持する |
| 後続の同時登録 | 日付と競合時間帯を含むエラーを表示する |
| 後続の同時登録後に下書き一覧を開く | `時間重複` と競合時間帯が表示される |
| 既存案件の時間を重複範囲へ変更する | 変更を拒否し、元の時間枠を維持して画面上の変更値を残す |
| 開始時間だけを変更して一時的に開始が終了以後になる | DBへ保存せず、終了時間も正常に変更された時点でまとめて保存する |
| `＊未入力` の通常案件と重複する | 時間枠確保済みとして重複エラーになる |

### スケジュール一覧

| ケース | 期待結果 |
| --- | --- |
| 日本時間でサイトを開く | 現在年月の一覧が表示される |
| 月タブを表示する | 前月・当月・翌月の3か月分が表示される |
| 対象月の一覧を表示する | その月の水曜日・金曜日が日付順に表示される |
| 一覧を表示する | 8:30-17:30が30分単位で表示される |
| 空白セルをクリックする | その日付の新規入力フォームが開く |
| 入力済みセルをクリックする | 対応する既存案件フォームが開く |
| 複数セルにまたがる案件を表示する | 先頭セルに依頼者名と作業種別、後続セルに矢印を表示する |
| 同じ日に複数案件を表示する | 開始時間順に5色を割り当てる |
| 6件以上の案件を表示する | 前後で同じ色が連続しないよう5色を再利用する |
| 入庫、商品管理を依頼者名なしで表示する | 先頭セルに作業種別だけを表示する |
| 今日より前の入力済みセルをクリックする | 閲覧専用で確認できる |
| 過去日へ変更リクエストを送る | 画面操作だけでなくService層でも拒否する |

### キャンセル

| ケース | 期待結果 |
| --- | --- |
| 依頼キャンセルを押す | 対象案件の情報を持つ確認ダイアログを表示する |
| 確認を取り消す | 案件を変更しない |
| キャンセルを確定する | 案件を物理削除し、該当セルを空きへ戻す |
| 下書きを削除する | 確認1回で物理削除する |
| 別画面ですでに削除された案件を更新する | `案件は削除されています` と赤字表示する |

## MVP後に追加するテスト

- 任意年月選択による過去・未来月への移動
- 祝日カレンダー同期、キャッシュ、取得失敗時の動作
- 祝日列を一覧生成時に除外し、案件データを自動削除しないこと
- 休み設定、解除、確認、対象案件の扱い
- 案件コピーとコピー先での重複チェック
- 通知、検索、バックアップなど、採用された将来機能
- 生成AIの使い方Q&A、確認付き入力支援、架空データによる試作、会社承認なしに実データを外部送信しないこと
- 架空データ限定の一時共有環境への複数端末接続、同時操作、再起動後のDBデータ保持
- 正式運用前のバックアップ取得と復元テスト

## テストデータ方針

テストデータは架空情報のみ使用する。設置、回収、交換、配達、入庫、商品管理を一通り含め、住所の市名は名古屋市、豊田市、岡崎市、一宮市などに限定する。番地、建物名、顧客名、担当者名は実在情報を使わない。

## CI方針

GitHub Actionsで次を自動実行する。

- ビルド
- 単体テスト
- Repositoryテスト
- Controllerテスト
- フォーマットチェック

フェーズ2ではMockMvcによる縦切りテストまでを追加し、実ブラウザを使うE2Eは未実装である。フェーズ4で主要経路のE2Eを追加し、ローカルでの安定性と実行時間を確認してからCI対象を決める。

## 手動テストと試運転

機能の正しさは配送・設置担当者とCodexでテストし、端末差と使いやすさは実機と利用者が確認する。

- 個人iPhoneで一覧と案件詳細の閲覧を確認する
- 社用スマートフォンで一覧と案件詳細の閲覧を確認する
- ノートPCで主要操作を確認する
- 架空データ限定の一時共有URLへ対象端末から接続する
- 共有環境のアプリ再起動後もDBデータが保持されることを確認する
- ブラウザの共有URLから利用し、PWAインストールを前提にしない
- セールス部門の数名がUIとUXを評価する
- 指摘修正後、セールス部長が導入可否を判断する

## 初期MVPの完了条件

- サイト起動時に現在月が表示される
- 前月・当月・翌月のタブを切り替えられる
- 水曜日・金曜日、8:30-17:30、30分単位の一覧が表示される
- 入力内容が自動保存され、未完成入力を下書き一覧から再開・削除できる
- 作業種別ごとの一覧反映条件を満たし、重複のない案件だけが `PUBLISHED` として一覧へ反映される
- 重複入力と同時登録を安全に拒否できる
- 同行条件、出庫状態、作業種別ごとの必須判定が動作する
- 二重確認後に案件をキャンセルできる
- 今日より前の日付を閲覧専用で確認できる
- iPhoneで一覧と案件詳細を閲覧し、ノートPCで主要操作を確認できる
- READMEの手順でテストを再実行できる

## 正式運用前の品質ゲート

初期MVPではバックアップ機能を完成条件に含めない。ただし、実在案件の入力を始める前に次を必須確認とする。

- 正式配置先とDBが会社の情報管理方針を満たす
- DBバックアップを取得できる
- 架空データのバックアップから復元できる
- 誤削除、障害時の最低限の対応手順と連絡先がある
- 上記の結果を記録し、正式運用の承認材料にする
